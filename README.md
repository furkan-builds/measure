# Measure

A calorie counting and nutritional tracking app with weight tracking. Targets web and mobile from a single monorepo.

## Tech Stack

- **TypeScript** — Statically typed superset of JavaScript. We use strict mode for maximum type safety across the entire codebase.
- **pnpm** — Fast, disk-efficient package manager. Its workspace feature lets multiple packages in this repo share dependencies and reference each other via `workspace:*` links.
- **Turborepo** — Build system for monorepos. It runs tasks (build, test, lint) across all packages in the correct dependency order, caches results, and parallelizes where possible. Config lives in `turbo.json`.
- **React** — UI component library. Used for both the web app and (via React Native) the mobile app.
- **Vite** — Frontend build tool and dev server for the web app. Extremely fast hot module replacement (HMR) during development. Config lives in `apps/web/vite.config.ts`.
- **Expo** — Framework and platform for React Native apps. Simplifies building, running, and deploying to iOS and Android without needing Xcode/Android Studio for most development.
- **Express** — Minimal Node.js HTTP server framework. Serves as the foundation for the API, with tRPC mounted as middleware.
- **tRPC** — End-to-end typesafe API layer. Defines API procedures (queries/mutations) on the server that the client can call with full TypeScript inference — no REST routes, no codegen, no OpenAPI spec needed. The server defines a router, and the client imports its type to get autocomplete and type checking.
- **PostgreSQL** — Relational database. Stores all persistent data (users, foods, weight logs).
- **Drizzle ORM** — Lightweight TypeScript ORM for PostgreSQL. Database tables are defined as TypeScript objects in `packages/db/src/schema/`. Migrations are auto-generated by diffing schema changes (`pnpm --filter=@measure/db generate`), not written by hand. Drizzle Studio provides a browser-based UI for inspecting the database.
- **Zod** — TypeScript-first schema validation library. Schemas in `packages/shared/` define the shape and constraints of data (e.g., "calories must be a non-negative number"). These schemas are used by tRPC for API input validation and by the frontend for form validation — single source of truth.
- **Biome** — Fast linter and formatter that replaces both ESLint and Prettier. Config lives in `biome.json`. Run `pnpm lint` to check and `pnpm lint:fix` to auto-fix.
- **Vitest** — Test runner built on Vite. API-compatible with Jest but faster due to Vite's transform pipeline. Used for unit and integration tests in web, api, shared, ui, and db packages.
- **Jest** — Test runner for the mobile app via the `jest-expo` preset, which handles React Native's Babel transforms.

## Repository Structure

```
apps/
  web/          — Vite + React web app
  mobile/       — Expo React Native app
  api/          — Express + tRPC API server
packages/
  shared/       — Shared Zod schemas and constants
  db/           — Drizzle schema, migrations, database client
  ui/           — Shared React component library
```

## Getting Started

### Prerequisites

- Node.js 22+
- pnpm 10+
- Docker and Docker Compose

### Setup

```bash
pnpm install
```

Start PostgreSQL:

```bash
pnpm docker:up
```

Create `apps/api/.env` (or copy from the example):

```bash
cp apps/api/.env.example apps/api/.env
```

Run database migrations:

```bash
pnpm --filter=@measure/db migrate
```

### Development

```bash
pnpm dev                    # run all apps
pnpm dev:web                # web only (http://localhost:3000)
pnpm dev:api                # api only (http://localhost:3001)
```

### Docker

```bash
pnpm docker:up               # start PostgreSQL in the background
pnpm docker:down             # stop PostgreSQL
docker compose down -v       # stop and wipe database data
```

### Other Commands

```bash
pnpm build                  # build all packages/apps
pnpm lint                   # biome check
pnpm lint:fix               # biome check --write
pnpm format                 # biome format --write
pnpm test                   # run all tests
pnpm test:web               # run tests for a specific package
pnpm typecheck              # tsc --noEmit across all packages
```

### Database

```bash
pnpm --filter=@measure/db generate   # generate migration from schema changes
pnpm --filter=@measure/db migrate    # apply pending migrations
pnpm --filter=@measure/db studio     # open Drizzle Studio (DB browser)
```

## Architecture

- **tRPC routers** in `apps/api/src/routers/` define the API. Both web and mobile apps consume the tRPC client, giving full type inference from DB schema through to the UI with no codegen.
- **Zod schemas** in `packages/shared/src/schemas/` are the single source of truth for validation — used by tRPC input validation and client-side form validation.
- **Drizzle schema** in `packages/db/src/schema/` defines all database tables. Migrations are generated from schema diffs, not hand-written.
- **Packages use `exports` maps** — import from specific paths (e.g., `@measure/shared/schemas/food`), not from a package root.
